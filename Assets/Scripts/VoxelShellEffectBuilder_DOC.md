# 体素透明外壳效果技术文档

## 目标效果

该效果用于表现“很多小方块拼接成一个大模型”的结构，但最终显示时不是逐块渲染，而是生成一个合并后的外壳：

- 只显示外露表面，内部相邻小方块之间的面全部剔除。
- 外壳面使用透明材质。
- 模型真实硬边使用黑色线条 Mesh 表现。
- 外露面上的单位网格交点生成黑点标记。
- 结果可在编辑器中预生成并保存为 Prefab，运行时直接加载。

当前实现入口为 `VoxelShellEffectBuilder`，外壳面生成依赖现有 `GreedyMesher`。

## 整体生成流程

输入数据是三维体素数组：

```csharp
bool[,,] voxels;
```

`true` 表示该位置有一个小方块，`false` 表示空。

执行 `Build Effect` 后生成三个子物体：

- `ShellSurface`：透明外壳 Mesh。
- `ShellEdges`：黑色边线 Mesh。
- `SurfaceDots`：黑色点标记 Mesh。

生成顺序：

1. 从 `VoxelChunk.voxels` 读取体素数据；如果没有可用数据，则生成默认楼梯测试数据。
2. 调用 `GreedyMesher.Build(voxels)` 生成透明外壳面。
3. 从体素外露面统计真实硬边，生成边线 Mesh。
4. 从体素外露单位面的中心采样，生成朝外的黑色面片 Mesh。
5. 挂载对应 `MeshFilter`、`MeshRenderer` 和材质。

## 外壳面生成算法

外壳面由 `GreedyMesher` 生成。

核心判断是：一个体素面只有在它相邻方向为空时才生成。

例如体素 `(x, y, z)` 的右侧面：

```text
voxels[x, y, z] == true
voxels[x + 1, y, z] == false 或越界
```

满足时右侧面是外露面，需要生成；否则它是内部接触面，直接剔除。

`GreedyMesher` 会进一步把同一平面、同一法线、连续相邻的小面合并成大矩形面，减少三角形数量。

关键点：

- 内部面在生成阶段就不会进入 Mesh。
- 大平面会被合并，渲染更轻。
- 顶点复用按“位置 + 法线”进行，避免硬边法线被错误平滑。

## 边线 Mesh 算法

边线不能直接从合并后的 Mesh 三角形边界提取。

原因是贪心合并会产生不同大小的共面矩形，矩形之间可能出现 T-junction。若从三角形边界提取，就会把同一个平面内部的矩形边误判为可见边，出现多余黑线。

当前正确做法是从体素外露面统计单位边。

### 单位边统计

遍历每个实心体素的六个方向：

1. 判断该方向是否是外露面。
2. 如果是外露面，取这个面的四条单位边。
3. 用边两端点的量化坐标作为 `EdgeKey`，把同一条空间边合并统计。
4. 记录该边被哪些面法线使用。

边的可见性规则：

- 只被一个外露面使用：外轮廓边，需要生成。
- 被多个外露面使用，且法线不同：硬边，需要生成。
- 被多个外露面使用，但法线相同：共面内部边，不生成。

这条规则能过滤掉同一个大平面里的多余线，只保留真正的模型轮廓和折角。

### 线条几何

每条可见边会生成一段细长方柱 Mesh：

- 方向：边的起点到终点。
- 宽度：`edgeWidth`。
- 外偏移：`edgeOffset`，用于减少与透明面 Z-fighting。
- 偏移方向：该边关联外露面法线的平均方向。

最终所有边线合并到一个 Mesh，挂在 `ShellEdges` 上。

优势：

- 不依赖 `LineRenderer`。
- 可以保存进 Prefab。
- 不受相机朝向影响。
- 运行时 Draw Call 更稳定。

## 黑点面片 Mesh 算法

黑点表示每个外露单位面中心的标记。最终生成的不是三维小球，而是贴在表面上的小面片。

生成规则：

1. 遍历每个实心体素的六个方向。
2. 如果相邻方向为空或越界，该方向就是外露面。
3. 取该外露单位面的中心点。
4. 在中心点沿外露面法线轻微外推 `dotOffset`。
5. 生成一个正面朝外的小四边形面片。

每个黑点生成一个小四边形面片：

- 半径：`dotRadius`。
- 材质：黑色不透明。
- 朝向：面片正面沿对应外露面法线朝外。
- 偏移：沿外露面法线轻微外推 `dotOffset`，减少 Z-fighting。
- 所有黑点面片合并进一个 Mesh，挂在 `SurfaceDots` 上。

注意：当前黑点是“每个外露小方块面的中心点”。因为点在单位面中心，不在面边界上，所以不会生成在线条或模型硬边上。前后两个相反外露面会各自生成一个朝外面片。

## 材质与渲染

当前项目使用 Built-in Render Pipeline。

默认材质策略：

- `ShellSurface` 使用 Standard 透明材质，`renderQueue = 3000`。
- `ShellEdges` 使用黑色不透明材质，`renderQueue = 3100`。
- `SurfaceDots` 使用黑色不透明材质，`renderQueue = 3110`。

边线和点的队列高于透明面，便于显示在透明外壳之上。

## 编辑器预生成

组件提供三个右键菜单：

- `Build Effect`：生成 `ShellSurface`、`ShellEdges`、`SurfaceDots`。
- `Clear Generated`：删除生成结果。
- `Save Generated Prefab`：保存 Prefab，并把临时 Mesh/Material 保存为资源文件。

保存位置：

```text
Assets/SavedMeshes/
```

保存时会把运行时生成的 Mesh 和 Material 先转成 `.asset` / `.mat`，再保存 Prefab，避免关闭 Unity 后 Mesh 引用丢失。

## 关键问题与处理

### 1. 为什么不能直接显示所有小方块？

小方块数量多时，逐块渲染会产生大量 GameObject、MeshRenderer 和内部不可见面，渲染成本高，也容易透明排序混乱。

使用外壳 Mesh 后，只保留最终可见表面，结构更干净。

### 2. 为什么边线不用三角形边界？

合并后的大面可能由多个不同尺寸矩形拼成。三角形边界里会包含很多视觉上不应该出现的内部边。

所以边线必须从体素拓扑判断，而不是从最终 Mesh 三角形直接判断。

### 3. 为什么点从体素面生成，而不是从外壳 Mesh 生成？

外壳 Mesh 经过贪心合并后，一个大矩形可能覆盖很多单位格。如果只看 Mesh 顶点，只能得到大矩形角点，会丢失每个单位面的中心标记。

因此黑点从体素外露面采样，才能保留单位尺度信息。点直接放在外露单位面的中心，天然避开边线。

### 4. 为什么需要偏移？

透明面、边线、黑点处在相同几何位置时会产生 Z-fighting。

`edgeOffset` 和 `dotOffset` 会把线和点稍微推出外壳表面，减少闪烁。

## 可调参数

`VoxelShellEffectBuilder` 的主要参数：

- `surfaceMaterial`：透明面材质。
- `surfaceColor`：默认透明面颜色。
- `edgeMaterial`：边线材质。
- `edgeColor`：边线颜色。
- `edgeWidth`：边线 Mesh 厚度。
- `edgeOffset`：边线外偏移。
- `dotMaterial`：黑点材质。
- `dotColor`：黑点颜色。
- `dotRadius`：黑点半径。
- `dotOffset`：黑点外偏移。
- `useVoxelChunkIfAvailable`：优先读取同物体上的 `VoxelChunk.voxels`。

## 验证场景

建议用以下形状验证：

- 单个方块：出现 6 个透明面、12 条边，并在 6 个外露单位面中心生成黑点面片。
- 两个相邻方块：中间接触面不出现，接触处内部边不出现。
- 实心长方体：大平面中间没有多余线，每个外露单位面中心有黑点，不出现在边线上。
- 楼梯模型：台阶折角有线，同一平面内部无线。
- 凹形模型：凹进去但外露的面、边、点都正常显示。

## 后续可扩展方向

- 把黑点小方片改成圆片或贴图圆点，视觉会更接近真正的点。
- 给边线 Mesh 增加发光材质或流动 shader。
- 支持非 1 单位体素尺寸。
- 支持运行时增量更新局部区域，而不是整块重建。
- 对非常大的体素模型按 chunk 拆分 Mesh，避免单 Mesh 顶点数过大。
